# Phase 7 — Operator

Detailed work plan for the eighth phase of the Rust rewrite. Companion
to [`rewrite.md`](./rewrite.md); the high-level summary lives there.
Builds on the codec scaffolding from [`phase-1.md`](./phase-1.md), the
storage engine from [`phase-2.md`](./phase-2.md), the single-broker
server from [`phase-3.md`](./phase-3.md), auth from
[`phase-4.md`](./phase-4.md), the cluster + coordinator surface from
[`phase-5.md`](./phase-5.md), and the transactional surface from
[`phase-6.md`](./phase-6.md).

**Goal.** Port `archive/cmd/skafka-operator/` + `archive/operator/api/`
+ `archive/operator/controllers/` to `bins/skafka-operator` +
`crates/sk-operator-api` + `crates/sk-operator-controllers`. The
existing Helm chart (`deploy/helm/skafka/`) deploys the Rust operator
alongside the Rust broker into a `kind` cluster, the same CRDs the Go
operator wrote bit-identical YAML for are now generated by
`cargo xtask gen-crds` walking the `kube-derive`'d Rust types, and the
`KafkaTopic` / `KafkaUser` / `KafkaCluster` reconcilers materialise the
same `/data/__cluster/credentials.json` + `/data/__cluster/acls.json`
+ partition directories the Go side produces — so a broker started
against a PVC populated by either operator is indistinguishable on the
wire. The broker also gets its admin write-path handlers
(`CreatePartitions`, `IncrementalAlterConfigs`,
`{Describe,Alter}ClientQuotas`, `DescribeConfigs`) wired in this phase
so all of skafka's CRs can be edited via the AdminClient surface and
not just `kubectl`.

**Length.** ~3 weeks, single engineer. Workstream A (CRD types) blocks
B/C and the broker-side TopicCRWriter; B (helpers + sweep) blocks C
(reconcilers); D (admin handlers in `sk-broker`) lands in parallel
with C against A's stable type surface; E (`xtask gen-crds`) lands any
time after A; F (`bins/skafka-operator`) closes with the
`tokio::select!` startup; G is the multi-CR `kind` smoke that gates
the merge.

**Out of scope for Phase 7.**

- Cross-broker `WriteTxnMarkers` *dispatch* (gh #114). Phase 6 wired
  the receiver-side handler + the file-queue dispatcher (commit
  `011939f`); Phase 7 doesn't need any further txn work. Documented
  here only because the rewrite plan listed it as a Phase 7/8
  candidate.
- `DescribeTransactions` (65) / `ListTransactions` (66) admin
  surfaces. Phase 8 candidates.
- `KafkaUserGroup` and standalone `KafkaACL` CRDs. **Deliberately
  absent** — the gh #135 closure folded ACLs back inline on
  `KafkaUser.spec.authorization`. Don't re-introduce them.
- Apiserver-side `+kubebuilder:default` defaulting on `KafkaUser` ACL
  fields (`spec.authorization.type`, `spec.authorization.acls[].type`,
  `spec.authorization.acls[].resource.patternType`). **Deliberately
  absent** — the gh #137 closure documents that apiserver defaulting
  collides with ArgoCD's GitOps reconcile (the stored object grows
  fields the git source omits and ArgoCD reports permanent drift).
  Defaults live in operator-side `acl_to_entry` instead. Port the
  same pattern; do not annotate `schemars(default = "...")` on those
  fields. The rule is **specific to the ACL surface** —
  non-ACL fields that already have `default:` in the committed
  YAML (KafkaCluster's `spec.listeners.internal.port: 9092` and
  `spec.listeners.external.port: 9093`, verifiable at
  `deploy/crds/skafka.io_kafkaclusters.yaml:88,127`) are
  apiserver-defaulted today and must stay that way — the chart's
  `templates/kafkacluster.yaml` relies on the apiserver filling
  them in when absent. Annotate those fields with
  `#[schemars(default = "default_internal_port")]` etc.
- Operator finalizers. The Go side deliberately doesn't use them
  (ArgoCD's parallel cascade-delete deadlocked when the operator pod
  went down before its CRs cleared). Rust port matches: cleanup runs
  reconcile-time on `Get → NotFound` + leader-elected startup sweep.
- `kube::runtime::leader_election` deep dive. Phase 7 uses
  controller-runtime's "manager Runnable + `needs_leader_election`"
  shape via `kube::runtime::controller`; if the kube-rs surface
  doesn't expose what we need, hand-roll a small Lease-patch loop
  matching the Go side (~80 LoC, same pattern Phase 5 used for
  `KubeLeaseElection`).
- Apiserver Admission Webhook (validating / mutating). The Go
  operator doesn't ship one; we don't either. Validation lives in
  `+kubebuilder:validation:*` attributes that `schemars` renders into
  the CRD's `openAPIV3Schema`.

**Prerequisite codec work.** Five new modules in `sk-codec/src/api/`:

| Key | API                       | Versions | Flexible from | Notes                                                                                          |
|----:|---------------------------|---------:|--------------:|------------------------------------------------------------------------------------------------|
|  32 | DescribeConfigs           | 0–4      | 4             | `resources[].resource_type` ∈ {2=Topic, 4=Broker, 8=BrokerLogger}; v3+ adds `documentation` field |
|  37 | CreatePartitions          | 0–3      | 2             | per-topic `count` + optional `assignments`; v1+ adds `validate_only`; v2+ flexible             |

After Phase 7 lands, `sk_codec::api::registry::ALL.len() == 29`
(Phase 6 ended at 24).

**Scope boundary (what real clients exercise).** `kafkactl`,
`kafka-configs.sh`, `kafka-topics.sh --alter --partitions`, and
`AdminClient.{createPartitions, incrementalAlterConfigs,
describeConfigs, describeClientQuotas, alterClientQuotas}` work
against the Rust broker. `kubectl apply -f kafka-topic.yaml` is
reconciled within ~1 s. `kubectl delete kafkatopic foo` removes
`/data/foo/` from the PVC and leaves no `.nfsXXXX` orphans (the broker
closed handles via the Phase 5 `TopicWatcher::Deleted` path). The
end-to-end kafka-compat suite under `archive/tests/kafka-compat/`
passes against the Rust operator + Rust broker pair, byte-equal
against the Go reference for `credentials.json`, `acls.json`, and the
`Status.TopicID` UUID format.

## Workstreams

Seven workstreams. A blocks B/C/D/E; B blocks C and unblocks F; C
blocks F; D lands in parallel with C against A; E lands any time after
A; F closes with the operator binary; G is the smoke that gates the
merge.

- **A** — `sk-operator-api`: CRD types with `kube-derive` + `schemars`
- **B** — `sk-operator-controllers` helpers: shared types, atomic
  writes, condition helpers, sweep functions
- **C** — `sk-operator-controllers` reconcilers: KafkaCluster,
  KafkaTopic, KafkaUser
- **D** — `sk-broker` admin handlers: keys 32, 37, 44, 48, 49 +
  `TopicCRWriter` (broker → CR patch path)
- **E** — `xtask gen-crds` + `xtask check-crd-drift`: schemars →
  YAML + Helm-chart mirror
- **F** — `bins/skafka-operator/src/main.rs`: tokio runtime,
  controller wiring, leader election, observability bootstrap
- **G** — Tests: per-reconciler unit tests, sweep tests, multi-CR
  `kind` smoke, CRD-drift CI gate, broker-side admin handler
  fixtures

Dependencies: A blocks everything else; B blocks C; C blocks F; D
parallel with C; E parallel with C; G blocked by F.

## A — `sk-operator-api`: CRD types

Port `archive/operator/api/v1alpha1/` into
`crates/sk-operator-api/src/`. Four CR types, one `lib.rs` that
re-exports them all. Each CR type uses `kube-derive`'s
`#[derive(CustomResource)]` to get the apiVersion / kind /
group / version metadata + a `K::api(client)` helper for free, and
`#[derive(JsonSchema)]` via `schemars` so workstream E's `gen-crds`
walker can emit the YAML.

### A.2 — Per-type shape

```rust
// crates/sk-operator-api/src/kafkatopic.rs
use kube::CustomResource;
use schemars::JsonSchema;
use serde::{Deserialize, Serialize};

#[derive(CustomResource, Debug, Clone, Serialize, Deserialize, JsonSchema, PartialEq)]
#[kube(
    group = "skafka.io",
    version = "v1alpha1",
    kind = "KafkaTopic",
    namespaced,
    status = "KafkaTopicStatus",
    derive = "PartialEq",
    printcolumn = r#"{"name":"Partitions","type":"integer","jsonPath":".spec.partitions"}"#,
    printcolumn = r#"{"name":"Ready","type":"string","jsonPath":".status.conditions[?(@.type=='Ready')].status"}"#,
)]
#[serde(rename_all = "camelCase")]
pub struct KafkaTopicSpec {
    #[serde(default, skip_serializing_if = "String::is_empty")]
    #[schemars(length(max = 249))]
    pub topic_name: String,

    #[schemars(range(min = 1))]
    pub partitions: i32,

    #[serde(default)]
    pub config: KafkaTopicConfig,
}

#[derive(Debug, Clone, Default, Serialize, Deserialize, JsonSchema, PartialEq)]
#[serde(rename_all = "camelCase")]
pub struct KafkaTopicConfig {
    #[serde(skip_serializing_if = "Option::is_none")]
    #[schemars(range(min = -1))]
    pub retention_ms: Option<i64>,

    #[serde(skip_serializing_if = "Option::is_none")]
    #[schemars(range(min = -1))]
    pub retention_bytes: Option<i64>,

    #[serde(skip_serializing_if = "Option::is_none")]
    #[schemars(range(min = 1))]
    pub segment_bytes: Option<i64>,

    #[serde(default, skip_serializing_if = "String::is_empty")]
    #[schemars(regex(pattern = r"^(delete|compact|compact,delete)?$"))]
    pub cleanup_policy: String,

    #[serde(skip_serializing_if = "Option::is_none")]
    #[schemars(range(min = 0))]
    pub min_compaction_lag_ms: Option<i64>,

    #[serde(skip_serializing_if = "Option::is_none")]
    #[schemars(range(min = 0))]
    pub delete_retention_ms: Option<i64>,
}

#[derive(Debug, Clone, Default, Serialize, Deserialize, JsonSchema, PartialEq)]
#[serde(rename_all = "camelCase")]
pub struct KafkaTopicStatus {
    #[serde(default, skip_serializing_if = "is_zero_i32")]
    pub partition_count: i32,

    #[serde(default, skip_serializing_if = "String::is_empty")]
    pub topic_id: String,

    #[serde(default, skip_serializing_if = "Vec::is_empty")]
    pub conditions: Vec<Condition>,
}

impl KafkaTopic {
    /// Mirrors `kafkatopic_types.go:EffectiveTopicName()`. Falls back to
    /// `metadata.name` when `spec.topicName` is empty — admin-protocol
    /// creates (gh #51 / gh #86) synthesise a hash-derived metadata.name
    /// and store the real Kafka name in `spec.topicName`.
    pub fn effective_topic_name(&self) -> &str {
        if self.spec.topic_name.is_empty() {
            self.metadata.name.as_deref().unwrap_or("")
        } else {
            &self.spec.topic_name
        }
    }
}
```

`Condition` is the apimachinery-equivalent in Rust. `kube` already
re-exports `k8s_openapi::apimachinery::pkg::apis::meta::v1::Condition`;
re-use it (with `#[derive(JsonSchema)]` already provided through
`k8s-openapi`'s `schemars` feature — verify before wiring; if missing,
hand-roll a `Condition` with the same JSON shape).

### A.5 — Schema generation

`schemars` ≥ 0.8 supports `derive(JsonSchema)` over the structs above.
`kube-derive`'s `crd()` associated function returns a
`CustomResourceDefinition` built from the schemars schema, which
workstream E (`xtask gen-crds`) walks to emit YAML.

`schemars` field attributes used:

- `#[schemars(range(min = N))]` — equivalent to
  `+kubebuilder:validation:Minimum=N`.
- `#[schemars(length(max = N))]` — equivalent to MaxLength.
- `#[schemars(regex(pattern = "..."))]` — equivalent to
  `+kubebuilder:validation:Enum=foo;bar` (rendered as a regex over
  the union).
- `#[schemars(description = "...")]` — equivalent to Go field
  comments. Carry over the Strimzi-divergence comments verbatim so the
  generated CRD docs read the same.

The Go side uses `controller-gen` to render `+kubebuilder:default=...`
into `default:` JSON-schema fields. **We mostly don't** — per the
gh #137 note, apiserver defaulting on the ACL surface collides with
ArgoCD; those defaults live in operator-side code (workstream B's
`aclToEntry`-equivalent). The non-ACL exception is the two
`KafkaCluster.spec.listeners.{internal,external}.port` fields which
ship `default: 9092` / `default: 9093` today and the chart's
`templates/kafkacluster.yaml` relies on the apiserver filling them
when absent — annotate those with `#[schemars(default = "...")]`
helpers and verify the rendered YAML still carries the `default:`
field via the golden fixture in workstream E.

**Atomic write.** `sk-storage::atomic_write::atomic_write_json` already
lives at the right altitude; re-export and use it. The Go side's
`writeAtomic` uses `json.MarshalIndent`; Rust uses
`serde_json::to_string_pretty` with the same 2-space indent so a
Rust-written `credentials.json` is byte-equal to the Go output for the
same input. Capture a Go-written `credentials.json` as a fixture
under `crates/sk-operator-controllers/tests/fixtures/` and assert
round-trip byte-equal — same trick the Phase 4 / Phase 5 / Phase 6
ports used.

**SCRAM derivation.** `pbkdf2 = "0.12"` lands in workspace deps;
re-use the `hmac` / `sha2` already shipped from Phase 4. The
`compute_scram(password) -> Result<ScramCredential>` function:

```rust
let mut salt = [0u8; 16];
OsRng.fill_bytes(&mut salt);
let salted_pw = pbkdf2_hmac_array::<Sha512, 64>(password.as_bytes(), &salt, SCRAM_ITERATIONS);
let client_key = hmac_sha512(&salted_pw, b"Client Key");
let stored_key = sha512(&client_key);
let server_key = hmac_sha512(&salted_pw, b"Server Key");
Ok(ScramCredential {
    salt:        BASE64.encode(salt),
    stored_key:  BASE64.encode(stored_key),
    server_key:  BASE64.encode(server_key),
    iterations:  SCRAM_ITERATIONS,
})
```

Same algorithm as `archive/operator/controllers/credentials.go:109-127`.

### B.2 — `acls.rs`

```rust
#[derive(Debug, Default, Serialize, Deserialize, PartialEq)]
pub struct AclFile {
    pub version: u32,
    #[serde(default)]
    pub acls: Vec<AclEntry>,
}

#[derive(Debug, Clone, Serialize, Deserialize, PartialEq)]
#[serde(rename_all = "camelCase")]
pub struct AclEntry {
    pub principal: String,
    pub resource: AclResource,
    pub operations: Vec<String>,
    pub permission: String,           // "Allow" | "Deny" (capitalised; broker matches case)
}

pub async fn reconcile_acls(client: &kube::Client, namespace: &str, data_dir: &Path)
    -> Result<(), ControllerError>
{
    let users: ObjectList<KafkaUser> = Api::<KafkaUser>::namespaced(client.clone(), namespace)
        .list(&ListParams::default()).await?;
    let entries: Vec<AclEntry> = users.items.iter()
        .filter(|u| u.metadata.deletion_timestamp.is_none())
        .flat_map(|u| {
            let principal = format!("User:{}", u.metadata.name.as_deref().unwrap_or(""));
            u.spec.authorization.iter()
                .flat_map(|a| &a.acls)
                .map(move |a| acl_to_entry(&principal, a))
        })
        .collect();
    let path = data_dir.join("__cluster").join("acls.json");
    sk_storage::atomic_write::atomic_write_json(
        &*sk_storage::RealFs::new(),
        path.parent().unwrap(),
        "acls.json",
        &AclFile { version: 1, acls: entries },
    )?;
    Ok(())
}

fn acl_to_entry(principal: &str, acl: &KafkaUserAcl) -> AclEntry {
    let pattern_type = match acl.resource.pattern_type.as_str() {
        "" => "literal".to_string(),
        s => s.to_string(),
    };
    let permission = if acl.kind.eq_ignore_ascii_case("deny") { "Deny" } else { "Allow" };
    AclEntry {
        principal: principal.to_string(),
        resource: AclResource {
            kind: acl.resource.kind.clone(),
            name: acl.resource.name.clone(),
            pattern_type,
        },
        operations: acl.operations.clone(),
        permission: permission.to_string(),
    }
}
```

Pattern + permission defaulting happens here — gh #137 contract:
defaults live in Rust, never in `+kubebuilder:default`.

### B.4 — `sweep.rs`

Port `SweepTopics` + `SweepCredentials`. Both are free functions, both
take `&kube::Client`, both walk the live CR set + a directory listing

```rust
pub async fn sweep_topics(
    client: &kube::Client,
    namespace: &str,
    data_dir: &Path,
) -> Result<Vec<String>, ControllerError> {
    let topics: ObjectList<KafkaTopic> = Api::namespaced(client.clone(), namespace)
        .list(&ListParams::default()).await?;
    let keep: HashSet<String> = topics.items.iter()
        .map(|t| t.effective_topic_name().to_string())
        .chain(std::iter::once("__cluster".to_string()))
        .collect();
    // walk data_dir, removeAll(dir/<name>) for each non-keep, non-".prefix" entry
    let mut removed = Vec::new();
    for entry in std::fs::read_dir(data_dir)?.flatten() {
        let name = entry.file_name().to_string_lossy().to_string();
        if !entry.file_type()?.is_dir() || keep.contains(&name) || name.starts_with('.') {
            continue;
        }
        std::fs::remove_dir_all(entry.path())?;
        removed.push(name);
    }
    Ok(removed)
}
```

```rust
    client: kube::Client,
    namespace: String,

impl TopicCRWriter {
    pub async fn expand_topic(&self, name: &str, new_count: i32) -> Result<(), TopicWriteError> {
        let patch = json!({"spec": {"partitions": new_count}});
        Api::<KafkaTopic>::namespaced(self.client.clone(), &self.namespace)
            .patch(name, &PatchParams::default(), &Patch::Merge(&patch)).await?;
        Ok(())

    pub async fn update_topic_config(&self, name: &str, ops: &[ConfigOp])
        -> Result<(), TopicWriteError>
    {
        let mut spec_config = serde_json::Map::new();
        for op in ops {
            match op.kind {
                AlterOp::Set    => { spec_config.insert(op.key.clone(), json!(op.value)); }
                AlterOp::Delete => { spec_config.insert(op.key.clone(), Value::Null); }
                AlterOp::Append | AlterOp::Subtract => {
                    return Err(TopicWriteError::UnsupportedOp(op.kind));
                }
            }
        }
        let patch = json!({"spec": {"config": spec_config}});
        Api::<KafkaTopic>::namespaced(self.client.clone(), &self.namespace)
            .patch(name, &PatchParams::default(), &Patch::Merge(&patch)).await?;
        Ok(())
    }
}
```

## Risks & mitigations

- **`kube-derive` CRD shape vs `controller-gen` shape.** The
  rewrite-doc risk register flagged this on day one. Mitigation: the
  Phase 7 PR captures a fixture of the Go-generated YAML, runs
  `cargo xtask gen-crds` against it, and a unit test asserts byte-
  equal. If `schemars` diverges from kubebuilder on
  `+kubebuilder:validation:Enum` (kubebuilder emits an OpenAPI
  `enum: [...]` array; schemars may emit a regex), add a
  `canonicalise_yaml` pass that normalises both forms.
- **`kube-derive` namespace + `printcolumn` attribute ergonomics.**
  `kube-derive` parses `printcolumn` from a JSON-string attribute,
  which is brittle vs Go's `+kubebuilder:printcolumn:name=Foo,type=...`
  multi-arg syntax. Mitigation: verify each CRD's `printcolumn`
  blocks land identically in the generated YAML; if `kube-derive`
  drops fields or reorders them, fall back to hand-writing the
  `additionalPrinterColumns` array in a post-process step inside
  `xtask gen-crds`.
- **`KafkaCluster` external-listener nested struct depth.** Six
  nested levels (KafkaCluster → spec → listeners → external → tls →
  certManager → issuerRef). `schemars` handles arbitrary nesting,
  but a single misplaced `#[serde(rename_all)]` at a parent breaks
  every descendant. Mitigation: declare `#[serde(rename_all =
  "camelCase")]` on every struct; the Go side spells out every JSON
  tag, so mirror that habit. Unit test serialises a fully-populated
  KafkaCluster and asserts the JSON keys match a captured fixture
  from `kubectl get kafkacluster -o json`.
- **`apply_unstructured` semantics on cert-manager Certificates.**
  The Go side uses an `applyUnstructured` helper that does Get + (if
  found) merge spec + update, or (if not found) set OwnerReferences
  + create. kube-rs's `Patch::Apply` (server-side apply) would be
  cleaner but requires a stable field-manager name and risks
  ownership transfer races vs cert-manager's controller. Mitigation:
  port the Get-or-Create pattern verbatim (same field shape, same
  ownership story). Server-side apply is a Phase 8 candidate, not a
  Phase 7 gotcha.
- **`generateTopicUUID` output format drift.** The Go side hand-
  formats a v4-shape UUID; `uuid::Uuid::new_v4()` returns a different
  in-memory layout but the same string format. Mitigation: a unit
  test asserts the output matches the regex `^[0-9a-f]{8}-…$` and
  the v4 + variant bits are correct; cross-check against 1000
  iterations of the Go output (capture once into a fixture).
- **Owner-reference cascade vs ArgoCD parallel delete.** The Go side
  deliberately doesn't use finalizers because ArgoCD's parallel
  cascade-delete deadlocked on operator-pod-down (CRs blocked on a
  finalizer the operator wasn't there to drain). Rust port has the
  same design constraint. Mitigation: don't add finalizers under any
  workstream; cleanup is reconcile-time on `Get → NotFound` plus the
  leader-elected startup sweep. Document the constraint in the
  Phase 7 PR description so a reviewer doesn't push back on the
  "missing" finalizer field.
- **`sweep_credentials` orphan-deletion blast radius.** A buggy
  sweep that mis-identifies a live user as an orphan deletes their
  credential entry, breaking auth. Mitigation: the sweep operates on
  the CR list (the source of truth), not on the credentials file;
  pruning means removing entries from the JSON whose username has no
  corresponding CR. Unit tests cover the round-trip; the sweep
  requires leader election so a buggy operator pod can't run it
  twice in parallel.
- **`TopicCRWriter` patch race with the operator reconciler.** The
  broker patches `spec.partitions` from one direction, the operator
  reconciles `spec` from the other. kube-rs's `Patch::Merge` is
  field-level, so two non-overlapping patches don't conflict — but
  the broker's patch fires the operator's watch, which then
  re-reconciles. Mitigation: the operator's reconcile is idempotent
  (mkdir is OK if exists, TopicID is preserved). Test: patch
  `spec.partitions = 6` from the broker side, assert the operator
  fires once and the partition directories grow.
- **`kube::Client::try_default()` in CI.** In-process unit tests
  use `wiremock` (no real kube). The `kind`-based smoke needs a
  `KUBECONFIG`. Mitigation: gate the `kind` smoke behind
  `--features kind-smoke` so PR CI doesn't pay the 2 min `kind
  create cluster` cost; `main`-push CI runs it. Same trick the Phase
  5 controller-failover suite uses.
- **`kube::runtime::controller` requeue semantics.** Go's
  controller-runtime requeues with exponential backoff on error;
  kube-rs requires the reconciler to return `Action::requeue(d)` or
  `Action::await_change()`. Mitigation: explicit table for each
  reconciler: happy path returns `Action::await_change()`, transient
  errors return `Action::requeue(Duration::from_secs(10))`,
  non-retrying errors (gh #120 SecretNotFound) set the condition and
  return `Action::await_change()` so the next CR edit retries.
- **`gen-crds` regression on `schemars` minor bumps.** A schemars
  upgrade could change the YAML output shape and break drift CI on
  unrelated PRs. Mitigation: pin `schemars` at workspace level (it
  already is via `[workspace.dependencies]`); document in the crate
  README that bumping `schemars` is a dedicated PR that re-runs
  `xtask gen-crds` and commits the diff.

- **`KafkaCluster.spec.replicas` drift vs Helm-owned StatefulSet.**
  The chart templates `replicas` into the KafkaCluster CR from
  `.Values.broker.replicaCount`; the same value separately
  templates the broker StatefulSet's `replicas`. A user
  `kubectl edit`'ing the CR without a `helm upgrade` creates new
  per-broker Services that point at non-existent ordinals.
  Mitigation: document in §C.3's module doc; add an
  observability check (Phase 8) for the
  "Service exists but selector matches zero pods" condition. Not
  a Phase 7 gate.

- **`+kubebuilder:default` on the two listener ports.** The
  committed CRD YAML sets `default: 9092` / `default: 9093` on
  KafkaCluster's internal/external listener ports. Schemars'
  default-rendering goes through `#[schemars(default = "fn_name")]`
  where `fn_name` must be a path to a fn returning the field type;
  this is a small but real ergonomic difference from kubebuilder's
  literal `+kubebuilder:default=9092`. Mitigation: declare two
  `const fn default_internal_port() -> i32 { 9092 }` helpers and
  reference them; capture a fixture diff so a future schemars
  upgrade that changes the `default:` field rendering (e.g. inline
  vs `$ref`) surfaces in `cargo xtask check-crd-drift`.
